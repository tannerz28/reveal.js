Vagrant

Introduction

What is Vagrant?

http://www.vagrantup.com/
  • Written by Mitchell Hashimoto
  • Command line tool
  • Written in Ruby
  • Automates VM creation
    • VirtualBox
    • VMWare
    • Hyper-V
    • Parallels
  • Create environments that are similar as possible or identical with production servers
  • Integrates well with configuration management tools
    • Shell
    • Ansible
    • Chef
    • Puppet
  • Runs on Linux, Windows, MacOS

Why use Vagrant?

  • Create new VMs quickly and easily
    • Only one command! 
      vagrant up
  • Keep the number of VMs under control
  • Reproducability
  • Identical environment in development and production
  • Portability
    • No more 4GB .ova files
    • git clone and vagrant up

Why React Native?

"You are a web developer, and want to do native"

Advantages

  • Native
  • Shared code between iOS and Android
  • More JS developers available than native developers
  • Faster iteration when developing (hot reloading, easier to reason about state)
  • Can prototype quickly
  • Not a WebView!
  • Asynchronous by default
  • Support for callbacks and promises

Disadvantages

  • Not as performant as a pure native application.
  • Fast paced community. Lots of change
  • ReactMethods must be void - They cannot return
  • Written twice for both platforms*
  • Documentation?
